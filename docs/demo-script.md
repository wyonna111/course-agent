# 答辩演示脚本（成员 B 准备，约 5 分钟）

> 答辩前与 A、C 彩排至少 1 次。括号内为操作提示。

---

## 0. 开场（30 秒）— 建议 B 讲

大家好，我们是「课内有据」小组。这是一个**课内资料优先、回答可溯源**的课程问答助手，解决复习时 AI 胡编、课件难找、小组难协作的问题。

线上地址：`________________`（填写 Streamlit 地址）

---

## 1. 上传与索引（45 秒）— 建议 A 操作

1. 打开左侧 **资料库** 标签。
2. 上传 1 份课程 PDF 或 PPT。（提前准备好，避免现场等太久）
3. 点击 **解析并加入索引**，等待完成。
4. 指出 **已索引文件** 列表和片段数量。

> 讲解词：系统按**页**建立索引，保证表格和选项不被截断。

---

## 2. 可溯源问答（90 秒）— 建议 A 操作，B 讲解

1. 在右侧输入一个**概念题**，例如：`UART 是什么？`（换成你们课件里确定能答对的题）
2. 展示回答末尾的 `[本地: 文件名 第 N 页]` 引用。
3. 展开 **课内参考资料**，展示送入模型的**整页正文**。
4. 强调：用户可以核对依据，而不是盲信 AI。

> 讲解词：先检索课件，再让大模型组织回答；相关度低时会拒答或联网。

---

## 3. 协作空间（60 秒）— 建议 C 讲解，A 操作

1. 切换到 **协作空间** 标签。
2. 填写昵称，点击 **创建协作空间**。
3. 展示邀请码和分享链接。
4. （若提前安排）让 B 或 C 用手机/另一浏览器打开链接加入。
5. 在协作空间资料库上传文件，说明**资料全员共享**。
6. 点击 **同步消息** 拉取对话。

> 讲解词：小组复习时一人上传、多人提问，对话和资料在同一空间。

---

## 4. 人工纠错（45 秒，可选）— 建议 C 讲解

1. 在某一回答的参考资料下，展开 **纠错此页课件**。
2. 修改错误内容并保存。
3. 用同一问题再问一次，说明纠错后检索与作答会优先采用修正版。

> 若时间紧可跳过，口播「支持按页纠错」即可。

---

## 5. 对比实验结论（30 秒）— 建议 B 讲

（引用 report.md 第四节）

与直接问 ChatGPT 相比，课内有据的优势是：
- 有**页码引用**，可回溯课件；
- **减少胡编**，答案锚定在已上传资料；
- 支持**小组协作**与**纠错**。
